# Claude Sonnet にお願い 03

## 4. レイアウトの作成

移行元の <https://computer-union.jp/> に似た Hugo　用のレイアウトを作成します。

### 4.1. ヘッダー

ヘッダーは以下の要素で構成します。

- サイトのタイトル「コンピュータ・ユニオン」
- 労働者供給事業 --> `/pages/4732/`
- 加入のご相談・お問い合わせ --> <https://computer-union.jp/?page_id=4650>
- 検索

加入のご相談・お問い合わせのリンク先はリリース時に `/contact/` に差し替えます。

移行元のヘッダーに設定している ENIAC の画像は含みません。シンプルなレイアウトとしてください。

### 4.2. フッター

移行元の "労働者供給事業 の案件の内容・条件の例は しごと情報 をご参照ください"
から "ヘッダの写真は ENIAC です。" までをフッターの内容とします。
Hugo の仕様として可能であればこれらの内容を Markdown 形式の `src/_footer.md` としてください。

### 4.3. トップページ

トップページの内容は `src/pages/4731.md` としてください。

### 4.4. 記事の目次

「ブログ」と「しごと情報」の目次は `/blog/` と `/job/` に表示します。
「ブログ」と「しごと情報」の最近の投稿の目次を `src/pages/4731.md` の `{{< latest-posts section="blog" count="5" >}}` と `{{< latest-posts section="job" count="5" >}}` を使用して表示します。

### 4.5. お問い合わせ

「お問い合わせ」のフォームは `/contact/` に表示します。
フォームは後で実装します。当面は仮に「その他のページ」と同様の HTML を出力してください。

### 4.6. meta 要素と script 要素

移行元にはソーシャルメディアに引用された場合のカードの表示のために meta 要素を使用しています。
それらは後で実装します。当面は仮にそれらの　meta 要素は無しとしてください。

移行元には　Yahoo 広告や Google アナリティクスなどの script 要素を使用しています。
それらは後で実装します。当面は仮にそれらの　script 要素は無しとしてください。

### 5.1. 追加の仕様(1)

ヘッダーとフッターの背景色と文字色は記事本文と同じにしてください。

表示幅が 640px 以上の場合、フッターは記事本文の右に表示して下さい。
表示幅が 640px 未満の場合、フッターは記事本文の下に表示して下さい。

### 5.2. 追加の仕様(2)

「ブログ」と「しごと情報」の目次は25個で分割して下さい。

「ブログ」と「しごと情報」の各記事の上下に、「次」と「前」のリンクを追加して下さい。
